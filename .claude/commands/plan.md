Before implementing $ARGUMENTS, create a mini plan:

1. **Goal** — what exactly are we building (one sentence)
2. **Files to create** — list new files needed
3. **Files to modify** — list existing files that change
4. **Dependencies** — external packages, APIs, or services needed
5. **Steps** — numbered implementation steps in order
6. **Risks** — what could go wrong or block us
7. **Estimated complexity** — simple / medium / complex

Keep the plan under 20 lines. This is a thinking step, not a document.
Ask: "Plan looks good? Say 'go' to start implementing."
